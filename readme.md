# Spark UEBA
## Назначение
<p>Обнаружение в базах данных событий не характерных для типичного поведения приложений и пользователей.</p>

## Описание
<p>Приложение написано на PySpark фреймворке, и не является полноценным UEBA т.к. в нем не осуществляется профилирование.</p>
<p>Приложение ориентировано на аналитику больших данных с событиями, в чем помогает Spark.</p>
<p>Приложение разделено на две части: одна забирает данные за месяц из базы данных формируя тяжелый DataFrame, по наименее плотному расписанию (напр. 1 раз в день когда ресурсы свободны), а вторая – забирает данные за 5 минут, по наиболее плотному расписанию.</p>
<p>Таким образом, легкие данные не задерживаются на executer'ах более 5 минут постоянно обновляясь, а тяжелые хранятся втечение 1 дня.</p>
<p>На случай если одна база данных недоступна, приложение делает запрос в другую, что немного повышает отказоустойчивость.</p>
<p>Файлы конфигурации служат для настройки подключений, а также для установки критериев определения событий как аномалий.</p>

## Требования
Стандартные библиотеки:
- os
- uuid
- urllib
- csv
- json
- glob

Кастомные библиотеки:
- pyspark
- findspark

<p>Наличие базы данных ClickHouse.</p>
<p>Версия Python >= 3.7

## Использование
- Заполнить конфигурационные файлы по образцу. Конфигурационные файлы лежат в './configs'.
- Поместить приложения в удобный scheduler, например Cron или AirFlow.
- Запускать с кастомными параметрами расположения мастера и драйвера или оставить локальный способ. Для запуска можно использовать 'spark-submit'.
